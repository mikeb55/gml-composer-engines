# Portable Compositional Principles

**Version:** 0.1.0  
**Source:** Extracted from existing engines

---

## From Wyble

| Principle | Description |
|-----------|-------------|
| line_independence | Each line behaves as independent melodic entity |
| implied_harmony_through_motion | Harmony emerges from line interaction, not block chords |
| anti_clutter_discipline | Avoid static filler; both voices must be melodically coherent |

---

## From Ellington

| Principle | Description |
|-----------|-------------|
| section_role_clarity | Each section has unambiguous role (lead, pad, background, etc.) |
| lead_background_contrast | Clear foreground/background distinction |
| call_and_response | Sectional dialogue; call/response logic |
| density_mapping | Density arc; sparse → medium → dense → tutti |

---

## From Big-Band Architecture

| Principle | Description |
|-----------|-------------|
| energy_ramps | Energy builds over form |
| climax_spacing | Climax placed strategically; not too early |
| release_before_peak | Release bars before peak intensity |
| entry_exit_logic | Clear section entry and exit |

---

## From Contemporary Counterpoint

| Principle | Description |
|-----------|-------------|
| voice_interaction | Voices interact; not isolated |
| motif_handoff | Material transfers between voices |
| phrase_overlap | One line continues while another resolves |
| contrary_oblique_motion | Favour contrary and oblique over parallel |

---

## Application

These principles should become **portable compositional principles**—available to any engine that declares them in its `active_primitives` or `style_constraints`. They are not engine-specific; they are style-agnostic best practices.
