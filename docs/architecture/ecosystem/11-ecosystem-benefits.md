# Ecosystem Benefits Summary

**Version:** 0.1.0

---

## How This Helps

### Current Engines

| Engine | Benefit |
|--------|---------|
| **Wyble** | Maps to CandidateComposition; uses shared evaluation; export contract; freeze controls for melody/lyrics. |
| **Ellington** | Section role clarity; density mapping; shared GCE; energy arc from big-band architecture. |
| **Big Band Architecture** | Energy ramps, climax spacing; shared evaluation; canonical sections. |
| **Contemporary Counterpoint** | Voice interaction, motif handoff, phrase overlap from shared primitives; portable principles. |
| **Arranger Assist** | Profile maps; shared evaluation. |
| **Selective Generation** | Canonical candidate; shared export. |

---

### Future Engines

- New engines can be added without architectural drift
- Shared primitives, intelligence layers, evaluation, export
- Canonical candidate schema reduces integration cost
- Compositional laboratory for regression testing

---

### Songwriting Engine

- songwriting_intelligence_layer as default
- lyric-driven composition; hooks; prosody; vocal targets
- Freeze/rewrite controls (freeze_melody_rewrite_lyrics, etc.)
- anti_cliche_filter, imagery_bank, subtext_mode

---

### Jazz-Composer Systems

- jazz_composer_intelligence_layer as default
- motivic_development, phrase_rhetoric, harmonic_pacing
- Metheny/Scofield profile modules
- Crossover control when both songwriting and jazz active

---

### Orchestration Systems

- classical_songcraft_layer (Ellington-derived)
- section_role_assignment; density_orchestration_map
- energy_gradient; clarity_score
- Shared export contracts for Sibelius, ACE Studio

---

## Goal

Turn the whole ecosystem into a coherent compositional platform where:

- Improvements propagate system-wide
- New engines can be added without architectural drift
- Evaluation, repair, and export are shared and consistent
- Anti-mush and elegance standards apply everywhere
